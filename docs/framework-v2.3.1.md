Monetization Gate v2.3.1  
Dual Evaluation + Controlled Reframing Protocol

Step -1: Project Intent Declaration (Mandatory)

Primary intent (choose one):

A. Revenue-First Product  
Goal = generate direct, near-term revenue (SaaS, marketplace, consulting wrapper, etc.)

B. Strategic Substrate  
Goal = create reusable infrastructure, platform, runtime, protocol, or enabling layer

C. Signaling / Credibility Asset  
Goal = portfolio piece, resume builder, community proof, thought-leadership magnet

D. Experimental / Learning System  
Goal = skill/capability compounding, R&D, personal mastery

Optional secondary tag (recommended for hybrids):  
Example: Primary B + Secondary A (70/30 weight)

If intent is unclear or contested → stop and clarify before proceeding.

Evaluation mode follows the primary classification.

When to use each mode:
- Mode A: Use if primary goal is near-term cash flow (direct revenue soon).
- Mode B: Use if primary goal is long-term leverage / optionality creation.
- Use both (hybrid) when the project has meaningful revenue potential after verticalization.

Evidence Hierarchy (Core Rule – Applies to All Modes)

Tier 1 — Prepaid pilots / signed contracts / LOIs  
Tier 2 — Direct buyer interviews (≥5 credible conversations)  
Tier 3 — Strong public data + identifiable buyers / pricing signals  
Tier 4 — Conceptual / internal validation only  

Rule:  
If Tier ≤ 3 → strong evidence required for any economic or buyer claim.  
If Tier 4 → SRL (Substrate Runway Limit) applies in Mode B.

Mode A – Revenue-First Evaluation

Checklist  
- Trigger event / acute pain  
- Who loses money  
- Who risks job / reputation / compliance exposure  
- Who wastes measurable time (quantify hours/week if possible)  
- Budget authority & line item  
- Existing alternatives / competition  
- Switching cost (time, data, training, political)  
- Realistic ROI window & magnitude  
- Prepaid validation path (manual → LOI → pilot → prepaid)  
- Differentiation / moat strength  

Output Format  
- Revenue Confidence Score (1–10)  
- Primary Risk Factor  
- Fastest Validation Experiment (< 7 days)  
- Recommendation: Build / Validate More / Abandon

Mode B – Strategic Substrate Evaluation

Five dimensions (score each 0–10)

1. Monetizable Optionality  
   How many meaningfully different future verticals / use-cases does this enable?  
   (0 = single narrow use → 10 = broad reusable core infrastructure)

2. Integration Gravity  
   How sticky / high-switching-cost does adoption become once integrated?  
   (0 = trivial to replace → 10 = deep entanglement)

3. Verticalization Surface Area  
   How easily can this be specialized into compliance products, enterprise services, regulated offerings, consulting packages, vertical SaaS?  
   (0 = hard to specialize → 10 = many natural wrappers)

4. Defensibility Through Constraints  
   Does it gain advantage from air-gapped deployment, deterministic behavior, regulated environments, strong security posture?  
   (0 = generic infra → 10 = powerful regulatory/security moat)

5. Capability-Leverage Density  
   How much foundational capability / leverage does building this unlock?  
   (0 = incremental → 10 = unlocks new category of competence)

Substrate Score (1–10)  
Simple average of the five metrics.  
Optional weighting: Creator may assign 1.5× to one dimension (declare explicitly). Default = equal.

Interpretation Bands  
1–3 → Low leverage — likely distraction unless very low cost  
4–6 → Moderate optionality — must aggressively verticalize soon  
7–8 → Strong platform/substrate potential — worth continuing as core infrastructure  
9–10 → Foundational layer — high long-term option value  

If Tier 4 evidence → SRL default 4 weeks  
SRL adjustment: Solo or small-team founders may declare a reasoned extension (6–8 weeks) in Step -1, e.g., "limited hours per week" or "waiting on external dependency".

Controlled Reframing Probe

Activates if  
- Either score ≤ 6  
- OR creator explicitly requests reframing despite higher score  

Purpose  
Detect structural misframing — not to inflate weak ideas.

Rules  
- Maximum 4 reframing hypotheses  
- No invented buyers  
- No assumed budgets  
- No urgency inflation  
- No hypothetical market sizes without trigger  
- Each hypothesis must include:  
  - Structural shift  
  - Plausible buyer hypothesis  
  - Specific trigger event  
  - Concrete evidence required (within 2–4 weeks)  
- If no plausible reframing → state “No viable structural reframing found.”  
- Scores remain frozen until validation evidence is collected.

Reframing Hypothesis Template

Hypothesis #N  
- Structural Shift: [e.g. Product → Internal Tool / Different Buyer / Narrower Wedge]  
- Potential Buyer: [job title, industry — must be plausible]  
- Trigger Event: [specific operational moment]  
- Evidence Required: [interviews, pricing validation, workflow proof, etc.]  
- Validation Deadline: [≤ 4 weeks]

Hybrid / Dual Verdict

When Primary + Secondary declared:

Revenue Lens: [score] → [one sentence summary]  
Substrate Lens: [score] → [one sentence summary]  

If Controlled Reframing activated:

Reframing Status  
- Hypotheses generated: [N]  
- Evidence plan summary: [brief]  
- Scores frozen pending validation  

Overall Recommendation (choose one)  
- Continue + Verticalize (strong substrate, path clear)  
- Validate More (promising but evidence gap)  
- Pause / Reclassify (misframed or low leverage)  
- Sunset / Archive (no viable path on current axes)
